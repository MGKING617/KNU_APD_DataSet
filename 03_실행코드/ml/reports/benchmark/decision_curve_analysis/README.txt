# Decision Curve Analysis

이미지 파일: decision_curve_analysis.png

공통 벤치마크 기준
- 데이터: 지역사회건강조사 재범주화 데이터, 19세 이상 39세 이하 청년층 표본 134,672건.
- 참고 라벨: PHQ-9 합산점수 10점 이상을 Reference high, 10점 미만을 Reference low로 정의하였다.
- 클래스 분포: Reference high는 5,468건으로 전체의 4.1%이고, Reference low는 129,204건으로 전체의 95.9%이다.
- 벤치마크 목적: Logistic Regression, Random Forest, XGBoost, LightGBM을 같은 데이터 분할과 같은 전처리 조건에서 비교해 어떤 모델이 선별 목적에 적합한지 확인하는 것이다.
- 해석 주의: 벤치마크 결과는 모델 비교용이다. 실제 웹사이트에 연결된 운영 모델은 ml/artifacts/risk_model.joblib의 LightGBM이며 운영 임계값은 0.590이다. 벤치마크 LightGBM은 임계값 0.588 기준이므로 운영 모델과 매우 가깝지만 완전히 같은 산출물은 아니다.

그림이 보여주는 것
Decision Curve Analysis는 여러 임계확률에서 모델을 사용하는 전략이 Treat all 또는 Treat none 전략보다 얼마나 순편익이 있는지 보는 그래프이다. 의료 의사결정에서 모델의 실사용 이득을 보조적으로 평가할 때 사용한다.

벤치마크 용어 설명
- Threshold probability: 이 확률 이상이면 개입 또는 상담 권장을 하겠다고 보는 기준이다.
- Net benefit: TP로 얻는 이득에서 FP로 인한 손실을 가중해 뺀 값이다.
- Treat all: 모든 사람을 High처럼 다루는 전략이다.
- Treat none: 아무도 High로 다루지 않는 전략이다.

결과 해석
본 결과에서는 낮은 임계확률 일부 구간에서만 모델의 순편익이 나타나고, 임계값이 올라갈수록 Treat none보다 낮아지는 구간이 많다. 이는 현재 모델을 임상 개입 결정 도구로 직접 쓰기보다는 조기 상담 참고 신호로 해석하는 것이 적절하다는 뜻이다.

보고서에 넣을 문장
"Decision Curve Analysis에서는 모델 사용의 순편익이 제한적으로 나타났다. 따라서 본 모델은 치료 여부를 결정하는 임상 의사결정 도구라기보다, 상담 전 위험 신호를 확인하는 선별 보조 도구로 활용하는 것이 적절하다."

주의점
DCA는 임상적 손익 가정에 민감하다. 본 프로젝트처럼 고급프로그래밍설계 텀 프로젝트 기반 웹서비스에서는 DCA 결과를 과도하게 주장하지 않고 참고 분석으로 제시하는 것이 안전하다.
